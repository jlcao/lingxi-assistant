## 系统概述

### 1.1 项目简介

灵犀智能助手是一个基于 **Plan+ReAct 模式**的智能助手系统，采用 Python 后端引擎 + Vue+Electron 桌面前端的架构。系统支持多会话管理、长期记忆存储、技能扩展、工作区隔离等企业级功能。

### 1.2 核心特性

- **智能路由引擎**: 根据任务复杂度自动选择执行策略（直接回答 vs 计划执行）
- **完全异步化**: 所有核心组件支持异步操作，避免阻塞
- **多会话架构**: 支持多会话、多工作区隔离，支持断点重试
- **记忆系统**: 自动提取和存储用户偏好、重要事实，支持向量检索
- **事件驱动**: 发布-订阅模式，支持实时推送和多订阅者
- **安全机制**: 多层安全保护，包括沙箱、确认流程、命令白名单
- **SOUL 注入**: 工作区级别的个性化配置和身份定义
- **技能系统**: 动态技能加载、热重载、缓存优化

### 1.3 技术栈

| 层级 | 技术选型 |
|------|---------|
| **前端** | Vue 3 + TypeScript + Electron + Tailwind CSS |
| **后端** | Python 3.10+ + FastAPI + asyncio |
| **数据库** | SQLite (会话数据) + ChromaDB (向量存储) |
| **LLM** | OpenAI API (支持流式响应和思考模式) |
| **通信** | WebSocket + RESTful API |
| **日志** | Python logging + 自定义过滤器 |

## 核心功能模块

### 3.1 核心引擎模块 (core/engine/)

#### 3.1.1 BaseEngine (引擎基类)

**职责**: 提供引擎的公共功能和接口定义

**核心功能**:
- LLM 响应解析
- 行动执行
- 高危操作确认
- 事件发布
- 检查点管理

**关键方法**:
```python
- process(context: TaskContext) -> Union[str, Generator]
- _execute_action(action, action_type, action_input) -> str
- _handle_dangerous_operation_with_confirmation(action, parameters) -> Optional[str]
- _publish_task_start/end/failed(session_id, execution_id, ...)
```

#### 3.1.2 AsyncReActCore (异步 ReAct 核心)

**职责**: 实现 ReAct (Reasoning + Acting) 推理模式

**核心功能**:
- 异步流式 LLM 调用
- 思考过程实时推送
- 步骤执行循环
- 历史上下文构建

**执行流程**:
```
1. 构建初始消息 (系统提示词 + 历史上下文 + 可用技能)
2. 调用 LLM 生成思考和行动
3. 实时推送思考过程 (think_stream 事件)
4. 执行行动 (调用技能或工具)
5. 记录观察结果
6. 更新消息上下文
7. 重复步骤 2-6 直到完成
8. 发布任务结束事件
```

**关键方法**:
```python
- async _execute_task_stream(context: TaskContext) -> AsyncGenerator
- async _execute_step(step, messages, task_level, steps, context) -> Dict
- async _process_llm_response(messages, task_level, stream, thinking_mode) -> AsyncGenerator
```

#### 3.1.3 AsyncPlanReActEngine (异步 Plan+ReAct 引擎)

**职责**: 智能路由引擎，根据任务复杂度选择执行策略

**核心功能**:
- 任务分析和分类
- 计划生成
- 断点重试
- 智能路由 (直接回答 / 计划执行)

**执行策略**:
```
任务分析
    ↓
判断任务级别
    ├─ simple + 有直接答案 → 直接回答
    ├─ direct → 直接回答
    └─ complex + 有计划 → 计划执行
        ├─ 从检查点恢复 (如果存在)
        └─ 执行计划步骤
```

**关键方法**:
```python
- async _execute_task_stream(context: TaskContext) -> AsyncGenerator
- async _analyze_task_and_plan(context, history_context) -> Optional[Dict]
- async _execute_plan_steps(plan, context) -> AsyncGenerator
- async _resume_from_checkpoint(checkpoint, context) -> AsyncGenerator
```

```
  ├── step_id: str
```

#### 3.2.2 历史压缩机制

**压缩策略**:
1. **轻量级压缩** (默认):
   - 保留最近 10 轮完整对话
   - 旧对话移除详细步骤
   - result 字段截取前 200 字符

2. **LLM 智能压缩** (可选):
   - 使用 LLM 生成语义摘要
   - 保留关键信息
   - 适用于长对话历史

**触发条件**:
- 历史轮次超过阈值 (默认 50 轮)
- Token 使用量接近限制

### 3.3 记忆系统模块 (core/memory/)

#### 3.3.1 MemoryManager (记忆管理器)

**职责**: 管理长期记忆的存储、检索和提取

**核心功能**:
- 记忆存储 (MEMORY.md)
- 智能记忆提取
- 混合检索 (关键词 + 向量)
- 记忆分类管理

**记忆数据结构**:
```python
@dataclass
class Memory:
    id: str                    # MD5 哈希
    content: str               # 记忆内容
    category: str              # 分类: preference/fact/decision/todo/note
    tags: List[str]            # 标签
    importance: int            # 重要性 1-5
    created_at: float          # 创建时间戳
    access_count: int          # 访问次数
    workspace_path: str        # 所属工作区
    metadata: Dict[str, Any]   # 额外元数据
```

**检索机制**:
```
混合搜索 (HybridSearch)
    ├── 向量搜索 (VectorStore)
    │   └── ChromaDB 语义相似度
    └── 关键词搜索 (Database)
        ├── 内容匹配 (10分)
        ├── 标签匹配 (5分)
        ├── 重要性加权 (1-5分)
        └── 访问次数加权 (最多10分)

融合评分 = vector_weight * vector_score + (1 - vector_weight) * keyword_score
```

**关键方法**:
```python
- search_memory(query, category, tags, top_k, use_vector, vector_weight) -> List[Memory]
- save_memory(content, category, tags, importance, metadata) -> Memory
- load_memory(workspace_path) -> int
- save_to_file() -> None
```

#### 3.3.2 MemoryExtractor (记忆提取器)

**职责**: 从会话历史中自动提取重要记忆

**提取策略**:
- 使用 LLM 分析对话内容
- 识别用户偏好、重要事实、决策
- 自动分类和打标签
- 重要性评分 (1-5)

**触发时机**:
- 会话结束时自动提取
- 手动触发提取
- 定期批量提取

## 技术架构

### 4.1 设计模式

| 模式 | 应用场景 | 示例 |
|------|---------|------|
| **单例模式** | 全局唯一实例 | EventPublisher, MemoryManager, SkillSystem |
| **发布-订阅模式** | 事件系统 | EventPublisher + Subscribers |
| **工厂模式** | 技能加载 | SkillLoader |
| **策略模式** | 任务分类、执行模式选择 | TaskClassifier, ExecutionModeSelector |
| **观察者模式** | 会话存储订阅者 | SessionStoreSubscriber |
| **建造者模式** | 消息构建 | PromptTemplates |
| **模板方法模式** | 引擎基类 | BaseEngine |

### 4.2 异步处理

**完全异步化**:
- 所有 I/O 操作使用 asyncio
- 流式 LLM 响应处理
- WebSocket 实时通信
- 并行子代理执行

**异步优势**:
- 避免阻塞
- 提高并发性能
- 支持流式输出
- 更好的资源利用

### 4.3 持久化策略

| 数据类型 | 存储方式 | 用途 |
|---------|---------|------|
| 会话/任务/步骤 | SQLite | 结构化数据持久化 |
| 向量嵌入 | ChromaDB | 语义搜索 |
| 长期记忆 | MEMORY.md | 人类可读的记忆存储 |
| SOUL 配置 | SOUL.md | 工作区个性配置 |
| 检查点 | SQLite JSON 字段 | 断点重试 |

### 4.4 缓存机制

**多层缓存**:
1. **技能缓存**: SkillCache (TTL 300s)
2. **SOUL 缓存**: SoulCache (基于文件修改时间)
3. **提示词缓存**: PromptTemplates (enable_cache 参数)
4. **向量缓存**: ChromaDB 内置缓存

**缓存策略**:
- LRU 淘汰
- TTL 过期
- 手动失效
- 热重载

### 6.2 风险等级评估

**并发控制**:
- 子代理最大并发数: 5
- WebSocket 连接池
- 数据库连接池

### 7.3 数据库优化

**索引优化**:
```sql
CREATE INDEX idx_sessions_updated_at ON sessions(updated_at);
```

**查询优化**:
- 使用参数化查询
- 批量操作
- 事务管理

### 7.4 历史压缩
