# Lingxi Agent 终端助手界面设计文档（V2.2）
**技术栈**：Electron + Vue3 + TypeScript  
**核心变更**：全面采用 SSE 流式响应，实现前后端彻底解耦，优化实时通信体验，移除 WebSocket 依赖

## 版本历史
- **V1.0**：初始架构设计（PySide6）
- **V1.1**：引入 MVVM 架构与异步任务池（PySide6）
- **V1.2**：深度适配底层 Agent 能力（思维链可视化、断点续传、技能中心）（PySide6）
- **V1.2+**：交互深度优化与异常处理闭环（模型路由可视化、技能自愈、多断点管理、智能干预）（PySide6）
- **V1.3**：技术栈全面迁移至 Electron+Vue3+TypeScript，重构分层架构，优化跨平台一致性与开发体验，保留 V1.2+ 全量功能与验收标准
- **V2.0**：架构调整为纯客户端模式，删除主线程业务逻辑，通过 HTTP/WebSocket 与后端服务通信，实现前后端彻底解耦
- **V2.1**：根据后端 API 接口设计文档（V4.0）更新 HTTP 客户端、SSE 流式响应处理和类型定义，确保前后端数据模型完全对齐，**移除 WebSocket，改用 SSE 流式响应**
- **V2.2**：全面优化 SSE 流式通信机制，完善事件缓冲、心跳检测、自动重连策略，移除 WebSocket 相关代码，统一通信协议

## 一、设计核心目标（V2.2 架构优化）
基于 **Electron+Vue3** 实现**透明化、可控化、跨平台**的终端助手，采用**纯客户端架构**，通过**HTTP+SSE**与后端服务通信，彻底解决 V1.3 版本中前后端职责重叠、数据不一致、通信缺失等问题，同时完全保留 V1.3 的核心交互能力。

### 核心目标（全量继承 V1.3，架构层面重构）
1. **过程透明**：实时展示 Agent 思考路径、任务分级及模型路由策略（通过 SSE 接收后端流式推送）
2. **状态可控**：支持断点续传、步骤级重试、人工干预与多断点并行管理（通过 HTTP API 调用后端）
3. **能力可视**：动态展示技能加载/调用状态，提供异常诊断与自愈引导（通过 HTTP API 查询状态）
4. **资源感知**：实时监控 Token 水位、本地系统资源，支持精细化资源管理（通过 HTTP API 轮询）
5. **跨平台一致**：兼容 Windows/macOS/Linux，保证各平台交互体验无差异
6. **工程化高效**：基于 Vue3 组件化、Vite 构建、TypeScript 类型安全，提升开发与维护效率
7. **架构解耦**：桌面端专注 UI 展示与用户交互，后端服务处理所有业务逻辑，通过 HTTP/SSE 通信
8. **通信可靠**：SSE 连接支持心跳检测、自动重连、事件缓冲，保证实时性与稳定性

### V2.2 架构变更说明

**删除的模块**（V2.1 → V2.2）：
- ❌ `wsClient.ts`（WebSocket 客户端，完全移除，统一使用 SSE）
- ❌ 所有 WebSocket 相关的 IPC 处理器和事件监听

**优化的模块**（V2.1 → V2.2）：
- ✅ `sseClient.ts`：完善事件缓冲机制、优先级处理、心跳超时检测
- ✅ `apiClient.ts`：统一 SSE 配置管理，优化流式响应处理
- ✅ `index.ts`（主线程）：移除 WS 相关 IPC，简化为纯 SSE 通信
- ✅ `preload/index.ts`：移除 WS API 暴露，统一为 SSE 事件监听
- ✅ 所有 Vue 组件：更新事件监听，从 `ws:` 改为 `sse:` 前缀

**保留的模块**（V2.1 → V2.2）：
- ✅ `windowManager.ts`（窗口管理，系统级操作）
- ✅ `fileManager.ts`（文件操作模块，处理本地文件选择和打开）
- ✅ 所有 Vue3 组件（UI 展示与交互）
- ✅ Pinia 状态管理（前端状态缓存）
- ✅ IPC 通信（主线程/渲染进程通信）

### 2.1 工程化基础配置
- **代码规范**：ESLint + Prettier，统一代码风格
- **包管理**：pnpm，提升依赖安装速度，减小项目体积
- **预加载脚本**：preload.js，实现主线程/渲染进程安全通信，避免 Electron 安全风险
- **路由管理**：Vue Router（轻量），实现弹窗/面板的路由化管理
- **图标库**：Element Plus Icons + 自定义 SVG，统一图标风格

### 2.2 SSE 通信协议设计（V2.2 核心）

#### 2.2.1 SSE 事件类型定义
后端通过 SSE 推送以下事件类型，前端监听并更新 UI：

|事件类型|触发时机|数据内容|UI 响应|
|---|---|---|---|
|`task_start`|任务开始执行|`{execution_id, task, task_level, model}`|显示任务开始，展示任务分级和模型信息|
|`plan_start`|规划阶段开始|`{execution_id, task_id}`|准备显示规划过程|
|`think_start`|思考开始|`{execution_id, task_id, step_id, content}`|展开思维链面板，显示思考标题|
|`think_stream`|思考流式输出|`{execution_id, task_id, step_id, content}`|实时更新思考内容（打字机效果）|
