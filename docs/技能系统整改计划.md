# 灵犀助手技能系统整改计划

## 📊 现状分析报告

### 一、设计文档核心要求回顾

根据《技能系统详细设计文档.md》，核心要求包括：

#### 五级架构
- SkillSystem（全局入口）
- SkillLoader（加载器）
- SkillCache（缓存器）
- SkillRegistry（注册中心）
- Security（安全体系：分级沙盒 + 拦截器）
- ExecutorScheduler（执行调度器）

#### 核心标准类
- `SkillResponse`：统一标准返回结构
- `ExecutionContext`：全链路上下文
- `BaseSkill`：所有技能必须继承的基类

#### 关键能力
1. **分级安全沙盒**：L1（线程）/ L2（子进程+venv）
2. **防依赖冲突**：技能级 venv 虚拟环境
3. **防内存泄漏**：显式卸载模块 + sys.modules 清理 + GC
4. **统一返回规范**：SkillResponse 标准结构
5. **全链路追踪**：ExecutionContext 上下文
6. **异常转译**：不暴露原生堆栈
7. **MCP 协议映射**：内部适配器模式

## 🎯 整改计划

### Phase 1: 核心标准类（P0 - 必须完成）

#### 1.1 创建 `skill_response.py`
```python
class SkillResponse:
    success: bool
    code: int          # 200=成功 400=参数 403=安全 500=异常
    data: Any          # 技能返回
    message: str       # 提示/错误
    meta: dict         # skill_id, version, trace_id, cost_ms
```

#### 1.2 创建 `execution_context.py`
```python
class ExecutionContext:
    trace_id: str
    user_id: str
    skill_id: str
    trust_level: str
    permissions: list
    timestamp: float
    workspace: str
```

#### 1.3 创建 `base_skill.py`
```python
class BaseSkill:
    manifest: dict
    context: ExecutionContext
    async def execute(self, params: dict) -> Any
```

---

### Phase 2: 执行调度器（P0 - 必须完成）

#### 2.1 创建 `executor_scheduler.py`
- 系统技能池（线程池，高优）
- 第三方池（进程池，低优）
- L2 隔离池（独立 venv 进程）
- 统一异常转译

---

### Phase 3: 分级安全沙盒（P0 - 必须完成）

#### 3.1 增强 `security.py`
- `run_l1_sandbox()`：线程 + 钩子拦截
- `run_l2_sandbox()`：子进程 + 独立 venv
- 危险操作拦截：open/socket/subprocess/sys/eval/exec/import

#### 3.2 创建 `security_interceptor.py`
- `disclose_stage1()`：基础信息
- `disclose_stage2()`：能力参数
- `disclose_stage3()`：风险披露
- `require_confirm()`：用户确认
- `audit_log()`：安全审计

---

### Phase 4: 防内存泄漏（P1 - 重要）

#### 4.1 增强 `skill_cache.py`
- 热重载时强制清理 `sys.modules`
- 主动 `gc.collect()`
- LRU + TTL 自动回收

#### 4.2 增强 `skill_loader.py`
- `unload_module()`：显式卸载模块

---

### Phase 5: 技能级 venv（P1 - 重要）

#### 5.1 增强 `skill_loader.py`
- `has_virtual_env()`：检测 venv
- `create_venv()`：创建 venv
- `install_dependencies()`：安装依赖

---

### Phase 6: 集成与重构（P0 - 必须完成）

#### 6.1 重构 `skill_system.py`
- 集成新的标准类
- 集成执行调度器
- 集成分级沙盒
- 统一返回 `SkillResponse`

#### 6.2 重构 `action_caller.py`
- 使用 `ExecutionContext`
- 使用 `SkillResponse`
- 集成安全拦截器

## ⚠️ 高风险项说明

1. **兼容性**：重构会打破现有技能执行接口，需要迁移现有技能
2. **测试覆盖**：需要完善的单元测试和集成测试
3. **性能影响**：子进程隔离会带来一定性能开销
