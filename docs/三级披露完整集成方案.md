# 三级披露完整集成方案

## 📋 概述

本文档提供将三级披露（渐进式披露）完整集成到灵犀助手技能执行流程中的详细方案。

---

## 🎯 核心目标

1. **完整披露流程**：Stage1 → Stage2 → (Stage3 + 确认) → 执行
2. **信息可追溯**：披露信息附加到响应中，前端可展示
3. **用户可控**：高风险操作必须用户确认
4. **审计完整**：披露和确认信息记录到审计日志

---

## 🏗️ 架构设计

### 数据结构

```
SkillResponse.meta
  ├── disclosure: DisclosureInfo
  │     ├── stage1: Dict           # 基础信息披露
  │     ├── stage2: Dict           # 参数披露（脱敏）
  │     ├── stage3: Optional[Dict] # 风险披露（仅高风险）
  │     ├── needs_confirmation: bool
  │     ├── user_confirmed: bool
  │     └── confirmed_at: Optional[datetime]
  └── ...
```

```python
from dataclasses import dataclass
from typing import Optional
from datetime import datetime

@dataclass
class DisclosureInfo:
    """三级披露信息"""
    stage1: Optional[Dict[str, Any]] = None
    stage2: Optional[Dict[str, Any]] = None
    stage3: Optional[Dict[str, Any]] = None
    needs_confirmation: bool = False
    user_confirmed: bool = False
    confirmed_at: Optional[datetime] = None

class SecurityInterceptor:
    # ... 现有代码 ...
    
    def collect_disclosure(
        self,
        skill_id: str,
        skill_info: Dict[str, Any],
        params: Dict[str, Any],
        needs_confirmation: bool = False
    ) -> DisclosureInfo:
        """收集完整的三级披露信息
        
        Args:
            skill_id: 技能ID
            skill_info: 技能信息
            params: 技能参数
            needs_confirmation: 是否需要确认
            
        Returns:
            DisclosureInfo 实例
        """
        disclosure = DisclosureInfo(
            stage1=self.disclose_stage1(skill_id, skill_info),
            stage2=self.disclose_stage2(skill_id, params),
            needs_confirmation=needs_confirmation
        )
        
        if needs_confirmation:
            disclosure.stage3 = self.disclose_stage3(skill_id, params)
        
        return disclosure
```

### 2. 改进 SkillSystem.execute_skill

**文件**: `lingxi/skills/skill_system.py`

修改 `execute_skill` 方法，将披露信息附加到响应：

```python
def execute_skill(
    self,
    skill_id: str,
    params: Dict[str, Any] = None,
    pre_disclose_only: bool = False,
    user_confirmed: bool = False
) -> SkillResponse:
    """执行技能（统一入口，带安全检查）
    
    Args:
        skill_id: 技能ID
        params: 技能参数
        pre_disclose_only: 仅进行披露，不执行
        user_confirmed: 用户是否已确认（用于二次执行）
        
    Returns:
        SkillResponse 实例
    """
    if params is None:
        params = {}
    
    self.logger.debug(f"执行技能：{skill_id}")
    
    # 1. 检查技能是否存在
    skill_info = self.registry.get_skill(skill_id)
    if not skill_info:
        error_msg = f"技能不存在: {skill_id}"
        self.logger.warning(error_msg)
        return SkillResponse.error(
            message=error_msg,
            code=ResponseCode.NOT_FOUND,
            skill_id=skill_id
        )
    
    # 2. 检查技能是否启用
    if not skill_info.get("enabled", True):
        error_msg = f"技能未启用: {skill_id}"
        self.logger.warning(error_msg)
        return SkillResponse.error(
            message=error_msg,
            code=ResponseCode.FORBIDDEN,
            skill_id=skill_id
        )
    
    # 3. 创建执行上下文
    context = ExecutionContext(
        skill_id=skill_id,
        trust_level=skill_info.get("trust_level", TrustLevel.L1),
        workspace=str(self.sandbox.get_workspace_root())
    )
    
    # 4. 检查风险级别
    skill_risk = DangerousSkillChecker.check_skill_risk(skill_id)
    command_risk = RiskLevel.LOW
    
    if skill_id == "execute" and isinstance(params.get("command"), str):
        command_risk = DangerousSkillChecker.check_command_risk(params["command"])
    
    needs_confirmation = (
        skill_risk in [RiskLevel.HIGH, RiskLevel.CRITICAL] or
        command_risk in [RiskLevel.HIGH, RiskLevel.CRITICAL]
    )
    
    # 5. 收集披露信息
    disclosure = self.security_interceptor.collect_disclosure(
        skill_id, skill_info, params, needs_confirmation
    )
    
    # 6. 如果仅披露模式，直接返回
    if pre_disclose_only:
        response = SkillResponse.success(
            data={"disclosure": disclosure},
            message="披露信息已生成",
            skill_id=skill_id,
            trace_id=context.trace_id
        )
        response.meta["disclosure"] = disclosure
        return response
    
    # 7. 需要确认但未确认时，返回等待确认
    if needs_confirmation and not user_confirmed:
        response = SkillResponse.error(
            message="需要用户确认",
            code=ResponseCode.FORBIDDEN,
            skill_id=skill_id,
            trace_id=context.trace_id
        )
        response.meta["disclosure"] = disclosure
        response.meta["needs_confirmation"] = True
        return response
    
    # 8. 记录用户确认
    if needs_confirmation and user_confirmed:
        disclosure.user_confirmed = True
        disclosure.confirmed_at = datetime.now()
    
    # 9. 确定执行方式
    trust_level = skill_info.get("trust_level", TrustLevel.L1)
    executor_type = ExecutorType.THREAD
    priority = SkillPriority.HIGH
    
    if trust_level == TrustLevel.L2 or skill_info.get("isolated_env", False):
        executor_type = ExecutorType.PROCESS
        priority = SkillPriority.LOW
    
    # 10. 执行技能
    start_time = time.time()
    
    def execute_task():
        # ... 现有执行代码 ...
        pass
    
    try:
        future = self.executor.submit(
            execute_task,
            skill_id=skill_id,
            context=context,
            priority=priority,
            executor_type=executor_type
        )
        
        response = future.result()
        cost_ms = (time.time() - start_time) * 1000
        response.meta["cost_ms"] = cost_ms
        response.meta["disclosure"] = disclosure
        
        self.security_interceptor.audit_log(
            skill_id=skill_id,
            params=params,
            response=response,
            context=context
        )
        
        self.logger.debug(f"技能执行成功：{skill_id}")
        return response
        
    except Exception as e:
        error_msg = f"技能执行失败：{skill_id} - {str(e)}"
        self.logger.error(error_msg)
        
        error_response = SkillResponse.error(
            message=error_msg,
            skill_id=skill_id,
            trace_id=context.trace_id
        )
        error_response.meta["disclosure"] = disclosure
        
        self.security_interceptor.audit_log(
            skill_id=skill_id,
            params=params,
            response=error_response,
            context=context
        )
        
        return error_response
```

### 3. 增强 ActionCaller

**文件**: `lingxi/core/action_caller.py`

```python
def call_with_security_check(
    self,
    skill_name: str,
    action_type: str,
    parameters: Dict[str, Any] = None,
    require_confirmation: bool = False,
    pre_disclose_only: bool = False,
    user_confirmed: bool = False
) -> Dict[str, Any]:
    """调用技能（带安全检查）
    
    Args:
        skill_name: 技能名称
        action_type: 行动类型
        parameters: 技能参数
        require_confirmation: 是否需要确认
        pre_disclose_only: 仅披露不执行
        user_confirmed: 用户已确认
        
    Returns:
        调用结果
    """
    # ... 现有安全检查代码 ...
    
    if action_type == "tool":
        context = ExecutionContext(skill_id=skill_name)
        try:
            if skill_name in self.tool.tools:
                tool = self.tool.tools[skill_name]
                
                if skill_name == "file" and parameters.get("action") == "list":
                    parameters["operation_type"] = "list"
                
                # 模拟披露信息（工具的披露）
                skill_info = {
                    "name": skill_name,
                    "description": tool.description,
                    "trust_level": "L2" if skill_name in ["file", "command"] else "L1"
                }
                
                needs_confirmation = skill_name in ["file", "command", "execute"]
                
                disclosure = self.security_interceptor.collect_disclosure(
                    skill_name, skill_info, parameters, needs_confirmation
                )
                
                if pre_disclose_only:
                    return {
                        "success": True,
                        "result": {"disclosure": disclosure},
                        "result_description": "披露信息已生成"
                    }
                
                if needs_confirmation and not user_confirmed:
                    return {
                        "success": False,
                        "result": "需要用户确认",
                        "result_description": "需要用户确认",
                        "disclosure": disclosure,
                        "needs_confirmation": True
                    }
                
                response = self.tool_adapter.execute_tool_in_sandbox(
                    tool.execute,
                    parameters,
                    skill_name,
                    context=context
                )
                
                result = {
                    "success": response.success,
                    "result": str(response.data),
                    "result_description": response.message,
                    "disclosure": disclosure
                }
                return result
            else:
                error_msg = f"工具 {skill_name} 未注册"
                self.logger.warning(error_msg)
                return {"success": False, "result": error_msg}
        except Exception as e:
            error_msg = f"工具执行异常: {skill_name} : {str(e)}"
            self.logger.error(error_msg)
            return {"success": False, "result": error_msg}
    else:
        return self.skill_system.execute_skill(
            skill_name,
            parameters,
            pre_disclose_only=pre_disclose_only,
            user_confirmed=user_confirmed
        ).to_dict()
```

## 🔄 执行流程

### 普通技能（低风险）

```
1. 用户/LLM 调用 execute_skill(skill_id, params)
2. Stage1 披露：技能基础信息
3. Stage2 披露：参数信息
4. 不需要 Stage3 和确认
5. 执行技能
6. 返回结果，包含 disclosure 信息
```

### 高风险技能

```
1. 用户/LLM 调用 execute_skill(skill_id, params, pre_disclose_only=True)
2. Stage1 + Stage2 + Stage3 披露
3. 返回 disclosure 信息，标记 needs_confirmation=True
4. 前端展示披露信息，等待用户确认
5. 用户确认后，调用 execute_skill(skill_id, params, user_confirmed=True)
6. 执行技能
7. 返回结果，包含 disclosure 和确认信息
```

## 📝 注意事项

1. **向后兼容**：保持现有接口不变，新增参数可选
2. **性能影响**：披露信息收集不应该显著影响性能
3. **隐私保护**：Stage2 必须对敏感参数脱敏
4. **审计完整**：所有披露和确认都必须记录审计日志
5. **用户体验**：确认对话框应该清晰、简洁、易于理解
