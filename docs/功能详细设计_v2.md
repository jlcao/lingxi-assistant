## 1. 子代理调度系统

### 1.1 概述

子代理调度系统允许灵犀助手在执行复杂任务时自动创建多个子代理并行执行，提升任务执行效率。

**核心特性：**
- 复用现有 SessionManager 多会话架构
- 自动检测并行意图
- 智能任务分解
- 实时进度监控
- 结果自动聚合

### 1.2 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                    灵犀助手主进程                             │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │           SubAgentScheduler (子代理调度器)           │   │
│  │                                                     │   │
│  │  - spawn() - 创建子代理                             │   │
│  │  - parallel_execute() - 并行执行                    │   │
│  │  - get_task_progress() - 获取进度                   │   │
│  │  - list_tasks() - 列出任务                          │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  ┌──────────────────┐  ┌──────────────────┐                │
│  │  Session 1       │  │  Session 2       │                │
│  │  (子代理 1)      │  │  (子代理 2)      │                │
│  └──────────────────┘  └──────────────────┘                │

```

### 1.4 自动检测机制

#### 并行意图检测

**位置：** `lingxi/core/engine/base.py`

**检测规则：**

```python
def _should_use_subagent(self, task: str, context: dict = None) -> bool:
    # 1. 关键词检测
    parallel_keywords = [
        "并行", "同时", "一起", "分别", "各自",
        "多个", "几个", "每个", "各自",
        "分析...和...", "对比...和..."
    ]
    
    # 2. 任务复杂度（长度 > 200）
    if len(task) > 200:
        return True
    
    # 3. 多行任务
    if '\n' in task:
        lines = [l.strip() for l in task.split('\n') if l.strip()]
        if len(lines) > 1:
            return True
    
    # 4. 上下文标记
    if context and context.get("use_subagent") is True:
        return True
    
    return False
```

#### 任务分解

**分解策略：**

```python
def _decompose_task(self, task: str, context: dict = None) -> List[str]:
    # 1. 按换行符分解
    if '\n' in task:
        return [l.strip() for l in task.split('\n') if l.strip()]
    
    # 2. 按逗号/分号分解
    if ',' in task or ',' in task:
        return [t.strip() for t in re.split(r'[,]', task) if t.strip()]
    
    # 3. 按"和"、"与"分解
    if '和' in task or '与' in task:
        return [t.strip() for t in re.split(r'[和与]', task) if t.strip()]
    
    # 4. 默认不分解
    return [task]
```

#### 进度获取

```python
def get_task_progress(self, task_id: str) -> dict:
    """从事件历史获取进度"""
    task = self.get_task(task_id)
    
    # 从 SessionManager 获取事件历史
    events = self.session_manager.get_events(task.session_id)
    
    # 分析步骤事件
    step_events = [e for e in events if e.get("type") in ["step_start", "step_end"]]
    
    total_steps = len(step_events)
    completed_steps = len([e for e in step_events if e.get("type") == "step_end"])
    
    return {
        "task_id": task_id,
        "status": task.status,
        "progress": int((completed_steps / total_steps) * 100),
        "current_step": step_events[-1].get("step", "") if step_events else "",
        "total_steps": total_steps,
        "completed_steps": completed_steps,
        "logs": [f"{e.get('type')}: {e.get('step')}" for e in step_events[-5:]]
    }
```

### 1.6 结果聚合

```python
def _aggregate_subagent_results(self, results: List) -> str:
    """聚合子代理结果"""
    aggregated = []
    
    for i, result in enumerate(results, 1):
        if hasattr(result, 'result'):
            aggregated.append(f"【子任务{i}】\n{result.result}")
        elif isinstance(result, dict) and 'result' in result:
            aggregated.append(f"【子任务{i}】\n{result['result']}")
        else:
            aggregated.append(f"【子任务{i}】\n{str(result)}")
    
    return "\n\n".join(aggregated)
```

## 2. SOUL 注入系统

### 2.1 概述

SOUL 注入系统让灵犀助手能够读取并遵循 SOUL.md 文件，塑造独特的个性和行为风格。

**核心特性：**
- SOUL.md 解析为结构化数据
- 5 分钟 TTL 缓存机制
- 自动注入到会话系统提示词
- 支持工作目录隔离

### 2.2 SOUL.md 格式

**标准格式：**

```markdown
# SOUL.md - Who You Are

## Core Identity
- **Name:** 灵犀
- **Creature:** AI 助手
- **Vibe:** 温暖、专业、幽默
- **Emoji:** 🦋

## Core Truths
- Be genuinely helpful, not performatively helpful
- Have opinions
- Be resourceful before asking
- Earn trust through competence

## Boundaries
- Private things stay private
- Ask before acting externally
- Never send half-baked replies

## Memory
- 用户偏好：喜欢简洁的回复
- 当前项目：灵犀助手开发
- 重要事项：每天早上 7:30 接收财经简报

## Context
用户关心的项目、正在进行的工作等自由文本内容
```

### 2.4 系统提示词构建

```python
def build_system_prompt(self, base_prompt: str = "") -> str:
    """
    构建格式：
    
    [基础系统提示词]
    
    ---
    
    # 你的身份 (SOUL.md)
    
    ## 核心身份
    - Name: xxx
    - Creature: xxx
    - Vibe: xxx
    - Emoji: xxx
    
    ## 核心原则
    - 原则 1
    - 原则 2
    ...
    
    ## 边界
    - 边界 1
    - 边界 2
    ...
    
    ## 记忆与上下文
    [用户记忆和当前上下文]
    """
    sections = []
    
    if self.soul_data.get("identity"):
        sections.append("## 核心身份")
        for key, value in self.soul_data["identity"].items():
            sections.append(f"- {key.capitalize()}: {value}")
    
    if self.soul_data.get("core_truths"):
        sections.append("## 核心原则")
        sections.extend([f"- {truth}" for truth in self.soul_data["core_truths"]])
    
    # ... 其他章节
    
    return base_prompt + "\n\n---\n\n# 你的身份 (SOUL.md)\n\n" + "\n\n".join(sections)
```

class SessionManager:
    def __init__(self, config, workspace_path=None):
        # 初始化 SOUL 注入器
        self.workspace_path = workspace_path
        self.soul_injector = SoulInjector(self.workspace_path)
        self.soul_injector.load()
    
    def create_session(self, session_id=None, **kwargs):
        session = Session(session_id, ...)
        
        # 注入 SOUL 到会话
        if self.soul_injector.soul_data:
            system_prompt = self.soul_injector.build_system_prompt(session.system_prompt)
            session.system_prompt = system_prompt
        
        return session

### 3.1 问题分析

**优化前的问题：**
1. SkillCaller、BuiltinSkills、SkillLoader 三个单例互相依赖
2. 缺少技能缓存机制，每次调用都重新读取文件
3. 技能执行路径不统一
4. 错误处理不完善

### 3.2 优化方案

#### SkillSystem 统一入口

### 4.2 数据流

#### 子代理执行流程

```
1. 用户输入任务
        │
2. Engine 检测并行意图
        │
3. 分解任务为子任务
        │
4. SubAgentScheduler.spawn() 创建子代理
        │
5. 每个子代理独立会话执行
        │
6. 发布事件（task_start, step_*, task_end）
        │
7. 聚合所有子代理结果
        │
8. 返回给用户
```

#### SOUL 注入流程

```
1. SessionManager 初始化
        │
2. SoulInjector 加载 SOUL.md
        │
3. SoulParser 解析为结构化数据
        │
4. SoulCache 缓存（TTL 5 分钟）
        │
5. 创建会话时注入 SOUL 到系统提示词
        │
6. LLM 接收包含 SOUL 的提示词
        │
7. 按 SOUL 定义的风格响应
```
