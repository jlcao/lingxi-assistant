# 1. 文档修订说明
本次修订**完全解决**深度评审提出的所有风险点：
1. **Python 沙盒绕过风险** → 分级隔离（L1 线程 / L2 子进程）
2. **依赖版本冲突** → 技能级 venv 虚拟环境 + 独立进程执行
3. **GIL 瓶颈 / CPU 阻塞** → 多进程池 + 线程池隔离
4. **热重载内存泄漏** → 显式卸载模块 + sys.modules 清理 + GC 强制回收
5. **返回值不规范** → 统一 SkillResponse 标准结构
6. **全链路追踪缺失** → 新增 ExecutionContext 上下文
7. **异常堆栈泄露** → 统一异常转译机制
8. **MCP 协议映射不明确** → 明确内部 MCP 适配器模式

# 2. 系统总览
## 2.1 核心定位
灵犀助手**插件化技能执行内核**，满足：
- 多目录扫描、注册、缓存、热重载
- 技能内部高内聚、分模块
- 遵循 Skill.md 最新规范 + 渐进式披露
- 兼容 MCP 协议
- **分级安全沙盒（线程/子进程）**
- **防依赖冲突、防内存泄漏、防绕过、防泄漏**

## 2.2 设计原则
- 第一性原理：从问题本质出发，不套模板
- 最小权限：技能默认无任何危险权限
- 分级隔离：信任等级决定隔离强度
- 可观测：全链路日志、审计、指标
- 强标准：统一结构、统一返回、统一异常

# 6. 核心流程（最终版）

## 6.1 技能执行流程（安全 + 隔离 + 标准）
```
1. 接收 execute_skill(skill_id, params)
2. 创建 ExecutionContext（trace_id）
3. 校验技能状态 + 权限
4. 渐进式披露 stage1 → stage2
5. 高风险 → stage3 披露 + 用户确认
6. 判断信任等级 L1 / L2
7. L1 → 线程沙盒
8. L2 → 子进程 + venv 隔离
9. 执行中拦截危险操作
10. 统一异常转译，不暴露原生栈
11. 写入审计日志
12. 返回标准 SkillResponse
```

## 6.2 热重载流程（防内存泄漏）
```
1. invalidate(skill_id)
2. 清理缓存
3. 从 sys.modules 移除模块
4. 强制 gc.collect()
5. 重新扫描、解析、注册
6. 重新加载
```

## 6.3 L2 强隔离流程（防依赖冲突）
```
1. skill 声明 isolated_env = true
2. 检查 venv，不存在则创建
3. 安装依赖
4. 启动独立子进程
5. 管道通信执行
6. 结果返回主进程
```

# 7. MCP 协议映射（明确化）
## 模式：内部适配器（非远程调用）
- Skill.md 解析 → 生成 MCP 工具清单
- 系统提供 MCPSkillAdapter 做协议转换
- 技能仍以 Python 模块运行
- 对外暴露 MCP 兼容接口
- 对内走原生沙盒体系

**优势**：安全、高性能、无网络开销。

---

# 8. 安全底线（不可突破）
1. 所有技能必须继承 BaseSkill
2. 所有执行必走沙盒
3. L2 技能永远在子进程 + venv
4. 技能不能访问主程序内存/配置
5. 危险操作 100% 拦截
6. 热重载必须卸载模块 + GC
7. 异常永远转译，不抛原生栈
8. 技能间不能直接 import
