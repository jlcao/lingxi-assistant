# 记忆管理技能

## 存放目录
系统用户目录/.lingxi/memory/MEMORY.md
## 核心规则
- 仅存储非敏感个性化信息，条目格式统一、分类层级清晰，严禁破坏文件原有结构
- 禁止存储密码、验证码、身份证、银行卡、手机号、详细住址等隐私数据
- 所有记忆操作必须通过内置file工具完成，禁止直接读写文件
- 记忆分类固定，仅允许单条条目增删改，禁止删除整段分类、标题或整个文件

## 存储结构
MEMORY.md 固定模板
📖 个人记忆库
## 1. 用户记忆（user）- 长期个性化信息
说明：永久留存，用于定制化服务，格式：- [YYYY-MM-DD] 记忆内容 | 标签（可选）
### 1.1 生活偏好与习惯
### 1.2 基础个人信息（非敏感）
### 1.3 长期目标与需求
### 1.4 交互偏好与关键结论

## 2. 工作记忆（work）- 临时上下文信息
说明：短期留存，任务结束可清理，格式：- [YYYY-MM-DD] 临时内容 | 类型

## 3. 待办事项（todo）- 任务清单
说明：进度可追踪，格式：- [YYYY-MM-DD] 任务内容 | 截止时间 | 进度（未完成/进行中/已完成）

## 执行流程
1. 初始化校验：调用file.read()检查文件，不存在则新建标准模板
2. 查询记忆：file.search()按分类/关键词检索，支持过滤标签、限制返回条数（limit=N）
3. 新增记忆：读取全文→对应分类末尾追加标准条目（自动生成当前日期）→file.write()覆盖写入
4. 更新记忆：检索定位目标条目→替换内容→保留结构覆盖写入
5. 删除记忆：仅移除单条无效条目，同步刷新统计数据
6. 统计记忆：总条数=用户记忆+工作记忆+待办事项，分类展示条数

## 输出规范
- 必显统计：总记忆条数、各分类条数、本次操作结果
- 记忆列表按时间倒序展示，限条数则截取前N条
- 仅输出用户可读结果，禁止暴露工具日志、代码片段

## 异常处理
- file工具调用失败：提示“记忆操作异常，请重试”，终止流程
- 检测敏感内容：拒绝存储并告知违规信息类型
- 条目格式错误：自动修正为标准格式后再写入
- 无匹配记忆：反馈“未找到相关记忆”
